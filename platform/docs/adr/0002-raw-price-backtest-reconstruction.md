# ADR 0002: 使用无复权价格进行回测重构与分红拆分精细化建模

## 状态
已接受 (Accepted)

## 背景 (Context)
在早期的回测引擎设计中，平台采用后复权价格（HFQ）进行统一计算与核算。这带来了以下严重偏误：
1. **隐式复利与现金拖累失真**：后复权因子假定分红款在除息日自动按当时价格再投资于原证券，且没有账期滞后，导致在长周期回测中高估了资产真实本金的累计表现，并且隐藏了分红款闲置在账户中的现金拖累效应。
2. **交易执行不真实**：证券交易在实际中必须四舍五入到 lot_size（通常为100股）进行原始计价交易，而后复权价对数量和价格进行了乘数放大，导致买卖手数及现金流扣减严重失真。

因此，为了实现彻底、干净的精细化财务建模，必须对底层所有计算和执行层重构为**无复权价格**计价，并在回测中动态模拟分红的除权到账及标的拆分事件。

## 决策 (Decision)
我们对平台底层进行了彻底重构（路径一），不保留对旧复权价回测的兼容：
1. **数据载入与平滑隔离**：
   - [data.py](file:///D:/strategy/platform/src/platform_core/data.py) 中数据全部加载为原始（无复权）价格；
   - 另设 `Bar.adj_close = close * factor` 字段提供复权价格历史。
   - 所有内置策略只在提取历史价格计算方差、协方差和动量等信号时读取 `adj_close` 以屏蔽跳空噪声，而回测主干则完全在无复权价下工作。
2. **应收分红与账期模拟**：
   - 登记分红数据表 [platform_dividends.csv](file:///D:/strategy/platform/data/platform_dividends.csv)，记录各 ETF 历史 `ex_date`、`payment_date` 和 `dividend_per_share`。
   - 回测循环中，在除息日（Ex-date）将分红金额存入 `dividend_receivables` 待收账款（计入组合总资产）；在实际派发日（Payment Date）到期时转入可用现金，真实还原现金拖累。
3. **拆分数量折算**：
   - 登记折算数据表 [platform_splits.csv](file:///D:/strategy/platform/data/platform_splits.csv)。
   - 折算日盘前自动对持仓股数与成本进行等比例调整，保障估值衔接。
4. **清理失效缓存**：
   - 物理删除所有此前在复权价环境下生成的 backtest cache 缓存文件，以防数据污染。

## 后果 (Consequences)
- **回测收益回归真实**：各资产配置组合的历史累计回报因去除了复权隐式复利与现金拖累，均呈现出合理收缩，更加切合实盘资产计价表现。
- **交易手数合规**：所有买卖手数计算、Lot-size 限制、手续费扣除完全发生在原始价格层面，契合实盘申报规范。
- **系统缓存干净**：所有缓存文件已重构更新，规避了由于计价单位变化造成的错误读取。
