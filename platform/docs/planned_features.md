# Platform 未实现与待完善功能

本文只列仍有工程价值的计划项，不保留旧路线图和旧系统迁移历史。

## 正式研究敏感性入口

当前 `platform/scripts/run_sensitivity.py` 支持按交易日步长采样，默认 `--step 3`，并按配置里的回测区间运行。正式研究规则要求每 2 个自然月一个 `start_date`，且训练样本截止到 `2025-06-30`。

待完善：

- 为正式研究增加专用参数或脚本，例如 `--calendar-month-step 2`、`--end-date 2025-06-30`。
- 在 manifest 和报告中明确样本上限、采样规则、共同可用起始日和数据快照时间。
- 输出候选相对基线的排名稳定性，而不仅是单配置的指标序列。
- 覆盖 Sharpe、年化收益、最大回撤、换手、交易数、订单数、拒单数和待执行意图压力。

## ETF 篮子筛选衔接

ETF 筛选属于 `etf_selection/`，平台只负责执行生成后的平台配置。

待完善：

- 固化 `etf_selection/generated_configs/<timestamp>/` 到 `platform/configs/` 可执行配置的契约。
- 增加生成配置的校验：资产代码、共同历史、数据新鲜度、训练样本长度和平台 schema。
- 在平台实验报告中记录 ETF 篮子来源、生成时间、输入 universe 和冻结参数。
- 保持平台内核不依赖 `etf_selection/` 源码。

## 数据新鲜度与样本隔离自动化

当前数据同步、共同日期检查和回测入口已存在，但研究前的数据新鲜度与样本隔离主要依赖人工执行规则。

待完善：

- 在实验入口前自动检查所需 symbols 的最新本地日期是否不早于当前日期前 7 个自然日。
- 自动检查所有所需 symbols 是否对齐，并输出共同可用历史区间。
- 当数据同步发生后，标记旧 cache 失效，除非 cache 能证明使用同等或更新的数据快照。
- 禁止正式研究入口在候选冻结前读取 `2025-07-01` 及以后的指标。

## 缓存可追溯性

当前存在 `platform/results/backtest_cache/` 使用记录，但正式研究需要更强的 cache 证明。

待完善：

- cache key 纳入 symbols、config hash 或参数集、样本窗口、数据快照时间、代码版本和平台版本。
- cache manifest 中记录是否包含 `2025-07-01` 及以后数据。
- 研究入口拒绝复用包含最终测试样本或数据快照不匹配的 cache。

## 策略版本治理加固

SQLite 已有策略草稿、版本、引用和删除保护，但还缺少完整产品化治理。

待完善：

- 提供稳定 CLI 或 API 管理策略草稿、发布版本、引用关系和删除保护。
- 在每次回测 artifact 中保存策略源码 hash、参数 hash、配置 hash 和运行环境摘要。
- 明确已发布策略版本的修改流程：只能复制为新版本，不能原地修改。
- 对内置策略注册增加测试，防止 Failed 或 research-only 策略误留在 `BUILTIN_STRATEGIES`。

## 导入导出与 manifest 完整性

当前回测会写出主要 CSV、metrics、配置和 checkpoint，但完整自描述归档仍可加强。

待完善：

- 每次回测输出统一 `manifest.json`，记录平台版本、Python 版本、配置 hash、策略 hash、数据快照和运行时间。
- 明确 raw artifact 与 report artifact 的 schema 版本。
- 为 checkpoint 恢复一致性增加端到端验证命令。

## 更广资产类型

当前平台主要服务日频 ETF/多资产配置研究。

待完善：

- 场外基金：申购确认日、赎回到账、费用阶梯和净值可见性。
- 期货：合约乘数、保证金、换月、夜盘和交割规则。
- 海外资产：交易日历、币种、汇率和费用模型。
- crypto：7x24 日历、交易所 fee profile 和独立数据适配器。

这些扩展必须通过各自的 `AssetAdapter`、数据 schema 和执行模型接入，不能污染现有日频平台核心语义。
