# 基于动态跟踪误差限制与交易成本摊销的换手优化风险平价策略实验报告 (R034)

## 1. 实验背景与核心思路
传统的 L1/L2 换手惩罚策略往往在整个回测期内施加恒定的偏离惩罚，导致持仓权重过度“锁死”，在市场趋势明确或风险环境剧烈切换时出现调仓时滞。

为了在换手率削减与风险平价跟踪精度之间取得更优的平衡，本实验提出了一种基于主动跟踪误差的软约束调仓决策方案，引入两大核心机制：
1. **动态跟踪误差限制 (Dynamic Tracking Error Limit)**：
   计算当前实际权重 $w$ 与最新目标风险平价权重 $w^*$ 的年化跟踪误差 $TE_{curr} = \sqrt{(w - w^*)^T \Sigma (w - w^*)}$。我们允许跟踪误差在动态阈值 $TE_{limit}$ 内漂移，而 $TE_{limit}$ 根据短期与长期系统波动率的比值进行动态缩放：
   $$TE_{limit} = TE_0 \times \exp(\gamma \times (vol\_ratio - 1.0))$$
   在市场剧烈震荡时主动放大容忍门槛以提供换手阻尼；在市场平稳时收紧门槛以保证跟踪精度。
2. **交易成本摊销过滤 (Transaction Cost Amortization Filter)**：
   衡量调仓带来的风险暴露改善与所需交易费用之间的性价比。调仓所需双边摩擦成本为 $TC$。若不调仓，每日承担的主动偏离风险惩罚为 $RiskPenalty_{daily} = 0.5 \times risk\_aversion \times TE_{curr}^2 / 252$；若调仓，则将 $TC$ 在预期的持有期内进行日均摊销 $TC_{daily} = TC / amort\_days$。仅当偏离风险惩罚大于日均摊销交易成本时才允许调仓：
   $$TE_{curr} > \sqrt{ \frac{504 \times TC}{risk\_aversion \times amort\_days} }$$

## 2. 实验核心参数设置
- 基础年化跟踪误差限制 $TE_0 = 0.02$ (2%)
- 跟踪误差波动率敏感度 $\gamma = 1.2$
- 跟踪误差上限与下限：$[TE_{min}, TE_{max}] = [0.005, 0.06]$
- 风险规避系数 $risk\_aversion = 4.0$
- 交易成本摊销天数 $amort\_days = 60.0$
- 预估单边交易成本率 $trade\_cost\_rate = 0.0005$ (万分之五)
- 检测与再平衡频率：`rebalance_frequency: daily` (每日检测)

## 3. 样本切分与回测对照设计
- **隔离规则**：
  - 训练/研究样本：数据截至 `2025-06-30`（含），用于策略选参、对照以及起点敏感性测试。
  - 最终测试样本：自 `2025-07-01`（含）起的数据，用于最终一次性验收。
- **多重对照**：在平台已有的核心风险平价配置（国内滚动、LW协方差风险平价、国内EWMA、全球EWMA、美股混合及跨境纳指全天候）下对 Baseline 与 Candidate 策略进行交叉回测。
- **国债替换**：强制将所有配置中的 Ten-Year Bond (511260.SH) 替换为模拟30年国债期货（`511260_3X.SH`）。
- **三年历史约束**：跨境纳指全天候配置 `global_nasdaq_all_weather_ewma.yaml` 最早共同可用行情日期为 `2022-05-31`，至训练末期仅 748 个交易日（不满三年），已根据规约**直接拒绝 (Rejected)**，不输出候选配置。其余 5 个配置均满足三年以上样本长度。

## 5. 失败归因与技术诊断
分析 `nav.csv`、`trades.csv` 和 `positions.csv` 实际持仓路径发现，虽然 Candidate 策略在年化金额换手率（IS 平均下降约 25%）与调仓笔数（降低 40%-60%）上表现出优越的优化效率，但它致命的逻辑偏误在于**在防守端引入了严重的调仓滞后与钝化**：

1. **动态限制与摊销条件的“防御锁死”**：
   在单边大跌或资产波动率飙升期，虽然目标风险平价权重 $w^*$ 发生了剧烈改变（需大幅撤出权益资产、增加低波资产的防御性权重），但由于 $TE_{limit}$ 受系统波动率之比 $vol\_ratio = vol_{short}/vol_{long}$ 的飙升而主动膨胀，使得允许的跟踪误差上限变宽。同时，由于交易成本摊销门槛的存在，当前的年化跟踪误差 $TE_{curr}$ 在尚未累积超过交易费摊销折算的阈值 $TE_{amort}$ 时，调仓便无法被触发。
   这导致在股票大跌初期，策略为了节省万分之五的交易费用而被“锁仓”，无法及时调整到防守资产（如 511260_3X.SH），造成了严重的避险滞后。最终导致最大回撤的大幅扩大（如国内 EWMA 组合的最大回撤从 **-10.20% 严重恶化至 -19.33%**），夏普比率在样本内外的全线退化。
2. **路径高度依赖与起跑点断点敏感**：
   双重门槛的存在使得策略极易受到初始入场日的持仓状态和调仓时机的影响。微小的入场日期不同导致首期再平衡的跟踪误差和交易成本摊销差异，这会在后续的 daily 检测中由于非线性激活发生发散，使得各起跑点之间的表现标准差（Sharpe Std 达 0.37 - 0.92）极为剧烈，暴露了策略的高度不稳健性。

## 6. 实验结论与落地动作
- **结论判定**：**Failed (失败)**。策略虽然大幅降低了换手摩擦，但以牺牲防守端的灵敏度为代价，造成了样本内外的夏普全线退化、最大回撤的大幅扩大以及敏感性测试的完全失效。
- **物理处理动作**：**物理拒绝合入平台主干**。
  1. 从 `platform/src/platform_core/strategy.py` 中物理擦除 `RiskParityDynamicTECostAmortizedStrategy` 策略类代码和在 `BUILTIN_STRATEGIES` 中的注册。
  2. 拒绝提交任何使用该新策略的专属平台配置文件。
  3. 仅在此非基线研究成果登记表中归档本实验的详细数据及结论，以防未来重复踩坑。
