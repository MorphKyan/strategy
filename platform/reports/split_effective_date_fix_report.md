# 份额拆分生效日修复报告（split effective date fix）

> 日期：2026-07-05
> 类型：引擎正确性修复（非策略研究，无候选/参数选择）
> 分支：`codex/platform-dashboard-mvp`

## 一、结论

回测引擎此前在拆分**基准日**（`platform_splits.csv` 的 `split_date`）当天调整持仓数量，而行情价格一律要到基准日之后的**首个真实交易日**才反映除权。两者错位导致"份额已翻倍 × 价格未除权"的单日净值假尖峰。修复后拆分统一在基准日后首个真实行情日生效。

**隔离验证（同数据、同配置、仅代码差一处）显示：修复不改变任何交易与期末净值（总收益 120.6713% 分毫不差、50 笔交易完全相同），只矫正净值路径**——受污染的最大回撤、波动、Sharpe 恢复真实水平（见 §四）。

## 二、根因

两个各自合理的机制叠加：

1. `data.py` 的 `LocalCsvBarData.bars_on()` 对无行情日用前收合成停牌 bar（估值连续性，正确）；
2. `engine.py` / `sim.py` 在 `split_date` 当天就执行 `quantity *= ratio`。

对照 `platform/data/platform_splits.csv` 全部事件与各标的行情文件，确认数据源（finshare）规律为：**`split_date` 是基准日，价格在其后首个真实交易日才除权**。受影响事件：

| 事件 | 基准日行情 | 错位后果（修复前） |
|------|-----------|------------------|
| 512890 2021-10-22（1:2） | 停牌无行 | 持仓市值虚增 1 倍一天 |
| 513100 2022-01-13（1:5） | 停牌无行 | 持仓市值虚增 4 倍一天 |
| 513500 2022-03-29（1:2） | 停牌无行 | 持仓市值虚增 1 倍一天 |
| 510500 2022-08-26（1:1.1454） | **有成交但价格未除权** | 持仓市值虚增 14.5% 一天 |
| 其余 5 个事件 | 在数据起点之前 | 不影响现有回测 |

510500 的情形说明"只在停牌时出错"不成立，错位是数据源日期语义问题，必须按"首个真实 bar"规则处理。

| 文件 | 变更 |

| `platform/src/platform_core/corporate_actions.py` | **新增**共享模块：`load_splits()`（支持 `config["data"]["splits_csv"]` 覆盖，供测试注入）、`resolve_split_effective_dates()`（生效日 = `split_date` 后该资产首个真实行情日）、`apply_due_splits()` |

## 四、修复前后指标对比（隔离 A/B）

同一数据快照（2026-07-05 同步，行情至 2026-07-03）、同一配置（`baseline_r1_domestic_rolling.yaml`，篮子 510300/512890/518880/511260_3X）、同一样本窗（2019-02-28 → 2025-06-30）、default 滑点场景，仅代码差本次修复：

| 指标 | 修复前 | 修复后 | 说明 |
|------|--------|--------|------|
| 累计收益 | 120.6713% | 120.6713% | **分毫不差**（尖峰是一日往返，无期末影响） |
| 交易数 / 订单数 | 50 / 50 | 50 / 50 | 完全一致，未诱发额外调仓 |
| 拒单数 / 跳过数 | 0 / 2 | 0 / 2 | 一致 |
| 年化换手 | 20.50% | 20.51% | 差异来自平均市值分母微变 |
| **年化波动** | **10.61%** | **6.65%** | 单日 ±15% 假波动被剔除 |
| **最大回撤** | **-14.09%** | **-7.39%** | 修复前从幻影高点起算 |
| **Sharpe** | **1.31** | **2.08** | 随波动矫正 |

尖峰对照（2021-10-22/25 日收益）：修复前 **+15.40% / -13.43%**；修复后 +0.26% / -0.35%。修复后全样本单日绝对收益最大 2.45%，不再有 >3% 的跳变。512890 份额在复牌日 2021-10-25 才由 125,800 调整为 251,600（价格 1.639 → 0.801），市值连续。

因交易序列与期末净值完全不变，本修复不涉及策略/参数选择，**无需起始日期敏感性分析**；受影响的仅是历史指标的度量准确性。

## 七、说明与已知局限

1. **旧的 2026-06-20 基线产物（`baseline_r1_domestic_rolling_20260620_005115_795863`）不能与本次运行直接对比**：其后配置篮子已由 511260 换为合成杠杆 511260_3X，且数据快照不同。该目录按 Hard Rule 6 原样保留。其他持有 512890/513100/513500/510500 跨拆分日的历史回测产物中的路径指标（回撤/波动/Sharpe）同样偏悲观，引用时注意，如需精确值请用修复后代码重跑。
2. **已知局限（未修，保持改动最小）**：复牌日 bar 的 `limit_up`/`limit_down` 仍由拆前前收推算（如 512890 2021-10-25 的跌停价被算成 1.48，而实际成交价 0.80），若恰有订单落在复牌日可能被误判涨跌停而拒单。本篮子月度调仓落在复牌日的概率极低，三场景拒单数均为 0；后续如做高频调仓策略需先修此点。
3. 本次 `sync_all_market_data.py` 刷新了 `platform/data/*.csv`（每行 `updated_at` 变化导致 diff 巨大），**未随本修复提交**，是否入库行情快照由维护者按惯例单独决定。
