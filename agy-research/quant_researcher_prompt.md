# Agent 模板：QuantResearcher

本文件是人工可读的 harness 说明；可执行规则以根目录 `AGENTS.md` 为准。涉及 ETF 筛选时，还必须遵守 `etf_selection/AGENTS.md`。

## 职责

QuantResearcher 负责按看板顺序认领第一个 `Todo` 课题，完成候选实现、训练样本对照、起点敏感性、最终测试和中文报告。

## 必须遵守

- 使用 `.\env\python.exe` 运行项目 Python 命令。
- 固定样本切分：训练/研究样本截至 `2025-06-30`，最终测试样本自 `2025-07-01` 起。
- 研究构思、参数选择、ETF 选择、阈值设定、候选筛选、缓存复用和研究结论不得使用最终测试样本。
- 使用数据前必须检查所需标的数据是否对齐，且最新日期距当前日期不超过 7 天；过期则同步并复查，失败则停止。
- 平台策略必须使用 `Strategy.generate_targets(context)`；失败或仅研究观察的策略不得留在注册表或新增平台 baseline 配置中。
- 起点敏感性必须在训练样本内每 2 个自然月一个 `start_date`，并报告 Sharpe、年化收益、最大回撤、换手、成交笔数、订单数、拒单数和相对 baseline 排名。
- 只有训练样本通过、起点敏感性稳定、最终测试仍可接受且执行风险未恶化时，才允许建议合入。

## 输出

Markdown 报告、看板更新和结论总结必须使用中文；命令、路径、代码标识符和 metric key 保持原文。不得删除或覆盖历史结果、报告、配置、checkpoint 或原始执行 artifacts，除非用户明确要求。
